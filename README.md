# Palmetto Coast High Leadership Academy -- Case Study

**Type:** Concept / Spec Project

**Client:** High school leadership program (JROTC-sstyles organization)

**Role:** Design | Frontend Development | Information Architecture | System Design

## OVERVIEW

This case study explores the design and development of a structured, multi-page website for the Palmetto Coast High School Leadership Academy.

Unlike traditional small business websites, this project focuses on creating a system that allows non-technical users (specifically rotating student leadership) to safely update and maintain website content over time.

The goal is to provide a stable, professional, and easy-to-manage platform that supports both external communication and internal program organization.

## PROBLEM

Many school programs either lack a dedicated website or rely on fragile solutions such as social media pages or basic site builders with no long-term structure.

In programs where students are responsible for maintaining content, this creates several issues:

- No consistent structure for updates.
- Content becomes outdated or disorganized over time.
- Risk of breaking layout or design when editing.
- Lack of clear ownership or responsibility.
- No system for handling rotating student roles.

A website for a program like this must not only present information clearly, but also provide a controlled and sustainable way for content to be updated year after year.

## CHALLENGES

This project introduces a different set of challenges compared to traditional business websites. These challenges are:

- Content is managed by non-technical users (students).
- Editors rotate frequently (yearly turnover).
- Mistakes must not break the site.
- The system must remain stable long-term.
- Both internal and external audiences must be supported.

The solution must balance flexibility with control, allowing updates without sacrificing structure or reliability.

## CONSTRAINTS

The case study was design with realistic program constraints in mind:

- Non-technical content editors.
- Rotating leadership roles.
- Limited technical oversight.
- Need for long-term maintainability.
- Simple and safe update process.
- Clear separation between structure and content.

The system must prioritize stability, clarity, structure, and usability over complexity.

## DESIGN APPROACH

The design process focuses on clarity, structure, and usability for both students and external visitors such as parents, for example.

Key priorities include a clear navigation and page structure, strong visual hierarchy for readability, mobile-first design for accessibility, organized content sections for easy updates, and professional and trustworthy presentation.

The goal is to create a website that is easy to understand, easy to navigate, and easy to maintain.

## DEVELOPMENT APPROACH

The site is built using semantic HTML, CSS, and lightweight JavaScript, with a focus on separating static structure from dynamic content.

A lightweight, file-based content system is implemented to allow students to update specific sections of the site without modifying the core layout or design. This approach ensures that content can be updated safely, structure remains consistent, the site is maintainable long-term, and no external dependencies or complex systems are required.

## OUTCOME

The final result is a structured, multi-page website that supports both public-facing communication and internal program needs.

The system allows student leadership to post announcements, update events, maintain leaderhsip listings, and manage gallery content. All without risking the integrit of the site.

This provides a sustainable and scalable solution for programs that rely on rotating, non-technical contributors.
